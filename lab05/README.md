### 3. Вероятностные модели и программная реализация

#### 3.1. Алгоритм выбора числа (Метод кумулятивной вероятности)
В отличие от прямого выбора из списка, в работе реализован метод, имитирующий распределение на отрезке $[0, 1)$. 
Пусть $M = 37$ (количество секторов), тогда вероятность каждого исхода $p_i = 1/37$. Программа генерирует случайное число $\alpha \in [0, 1)$ и ищет интервал, в который оно попало.

**Математическая модель:**
$$P(X = k) \text{ если } \sum_{i=1}^{k-1} p_i \le \alpha < \sum_{i=1}^{k} p_i$$

**Фрагмент кода реализации:**
```python
m = len(self.wheel_order)
p_i = 1.0 / m                # Вероятность одного исхода
alpha = random.random()      # Случайная точка на отрезке [0, 1)
cumulative_p = 0.0

for k in range(m):
    cumulative_p += p_i
    if alpha < cumulative_p:
        win_num = self.wheel_order[k]
        break
```


#### 3.2. Модель «Да / Нет»
Для бинарного выбора используется функция сравнения с порогом $p = 0.5$. Это простейший случай распределения Бернулли, где вероятность успеха и неудачи равны.
```python
def get_yes_no(self, p=0.5):
    return "ДА" if random.random() < p else "НЕТ"
```

#### 3.3. Тригонометрическая синхронизация
Для того чтобы стрелка указывала точно на выбранное число, индекс выигравшего числа ($win\_idx$) пересчитывается в градусы:
$$\text{Финальный угол} = win\_idx \times \frac{360^{\circ}}{37}$$
Это гарантирует соответствие графического результата внутренним вычислениям.
