## Базовый датчик случайных чисел
### Реалихованный базовый датчик случайных чисел (Линейный конгруэнтный генератор)
Суть алгоритма заключается в использовании модульной арифметики для вычисления последовательности чисел.
Каждое следующее число $X_{n+1}$ вычисляется на основе предыдущего $X_n$ по формуле:

$$X_{n+1} = (a \cdot X_n + c) \pmod m$$

Где:
* **$X_0$ (Seed / Зерно):** Начальное значение, с которого начинается генерация.
* **$a$ (Множитель):** Константа, на которую умножается текущее число.
* **$c$ (Приращение):** Константа, которая добавляется после умножения.
* **$m$ (Модуль):** Максимальный диапазон значений (обычно степень двойки, например $2^{31}$).
```python
class LCG:
    def __init__(self, seed=42):
        self.state = seed
        self.m = 2**32
        self.a = 1664525
        self.c = 1

    def next(self):
        # Формула: X_{n+1} = (a * X_n + c) % m
        self.state = (self.a * self.state + self.c) % self.m
        # Приведение к интервалу [0, 1]
        return self.state / self.m
```
#### Почему в итервале [0,1]?
Поскольку результат операции $\pmod m$ всегда находится в диапазоне от $0$ до $m-1$, для получения значения $\alpha$ (как требует базовый датчик), результат делится на модуль:

$$\alpha = \frac{X_{n+1}}{m}$$

#### Почему  "Псевдослучайный"?
Как отмечено в презентации, такие датчики называются **псевдослучайными**, потому что:
* **Детерминированность:** Если ты знаешь $X_0$ и параметры ($a, c, m$), ты можешь предсказать все последующие числа.
* **Периодичность:** Рано или поздно последовательность начнет повторяться. Хороший датчик должен иметь очень большой период.
* **Воспроизводимость:** Это важное требование из твоего списка. Используя одно и то же «зерно» (seed), ты получишь идентичную последовательность чисел, что критично для отладки моделей.
