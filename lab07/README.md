# Отчёт по лабораторной работе

## Марковская модель погоды (непрерывное время)

### 1. Цель работы
Изучить принципы построения и анализа непрерывных цепей Маркова. Реализовать программное приложение для имитационного моделирования динамики состояний погоды с графическим интерфейсом для:
1. Моделирования переходов между тремя состояниями (ясно, облачно, пасмурно) на основе матрицы интенсивностей переходов.
2. Визуализации процесса в реальном времени.
3. Расчета теоретического стационарного распределения и сравнения его с эмпирическими данными.
4. Сбора и хранения статистики в формате `.csv`.

---

### 2. Описание приложения

#### 2.1. Модуль настройки и управления
Приложение позволяет пользователю интерактивно задавать интенсивности переходов в виде матрицы 3×3. Диагональные элементы рассчитываются автоматически как отрицательная сумма недиагональных элементов строки. Реализован регулятор скорости симуляции для визуального анализа процесса на разных временных масштабах (от 0.1x до 5.0x).

#### 2.2. Система визуализации и логирования
Интерфейс разделён на три функциональные зоны:
- **Информационная панель:** отображает текущее модельное время и текущее состояние системы.
- **График траектории:** ступенчатая диаграмма последних 40 переходов, позволяющая отследить динамику изменений состояний во времени.
- **Панель сравнения:** гистограмма, в реальном времени сопоставляющая накопленные относительные частоты пребывания в состояниях с теоретическим стационарным распределением.

Все данные автоматически записываются в файл `weather_analysis.csv`, который обнуляется при нажатии кнопки «Сбросить».

### 3. Вероятностная модель

#### 3.1. Непрерывная цепь Маркова
Математическая модель описывается множеством состояний S = {1, 2, 3} и матрицей интенсивностей переходов Q:

$$
Q = \begin{pmatrix}
-1.1 & 0.6 & 0.5 \\
0.5 & -1.0 & 0.5 \\
0.5 & 0.8 & -1.3
\end{pmatrix}
$$

где q_{ij} ≥ 0 при i ≠ j — интенсивность перехода из состояния i в состояние j, а диагональные элементы определяются как:

$$q_{ii} = -\sum_{j \neq i} q_{ij}$$

Время пребывания в состоянии i распределено экспоненциально с параметром λ_i = -q_{ii}.

#### 3.2. Алгоритм моделирования
На каждом шаге алгоритма:
1. Генерируется время пребывания в текущем состоянии: Δt ~ Exp(λ_i)
2. Выбирается следующее состояние j с вероятностями:

$$P_{ij} = \frac{q_{ij}}{\lambda_i}, \quad j \neq i$$

#### 3.3. Стационарное распределение
Теоретическое стационарное распределение π = (π₁, π₂, π₃) находится из системы линейных уравнений:

$$\begin{cases}
\pi Q = 0 \\
\sum_{i=1}^{3} \pi_i = 1
\end{cases}$$

Для решения в приложении используется метод решения СЛАУ: система Q^T π^T = 0 дополняется уравнением нормировки, после чего решается методом numpy.linalg.solve.
